# 👀 产品观察笔记 (Observations)

> 看到好用的 APP、或者生活中糟糕的设计，立刻记下来。  
> **核心思维：拆解 (Deconstruct)**

---

## 2026-01-30

### [公司报销系统]
- **现象**: 上传发票需手动选日期，OCR识别率低  
- **思考**: 当前流程效率低下，用户体验差，存在优化空间  
- **优化方案**: 可考虑引入更先进的文档理解模型提升自动化水平  
- **底层逻辑**: 用户对报销流程的“效率预期”与系统“自动化能力”之间存在落差  
  - [ ] 对应心理学原理: 认知负荷理论

### [LayoutLMv3票据识别微服务]
- **现象**: LayoutLMv3在文档理解任务中表现优异  
- **思考**: 基于LayoutLMv3构建本地票据识别服务可显著提升识别准确率  
- **优化方案**: 开发轻量级微服务集成到现有报销流程中  
- **底层逻辑**: 利用预训练多模态模型融合文本与布局信息，提升结构化文档解析能力  
  - [ ] 对应心理学原理: 自动化信任（Automation Trust）

## 2026-01-29

### [小红书]
- **现象**: “摇一摇”加好友的交互  
- **思考**: 不需要点开扫一扫，符合直觉，这应该是利用了陀螺仪；感觉可以借鉴到我们那个社交模块里  
- **优化方案**: 考虑在社交模块中引入类似的直觉化交互设计  
- **底层逻辑**: 通过物理动作映射数字行为，降低操作门槛，增强自然交互感  
  - [ ] 对应心理学原理: 具身认知（Embodied Cognition）

### [公司报销系统]
- **现象**: 上传发票需手动选日期，OCR识别率低  
- **思考**: 当前流程效率低下，用户体验差，存在优化空间  
- **优化方案**: 基于LayoutLMv3模型构建本地票据识别微服务，提升识别准确率和自动化程度  
- **底层逻辑**: 用户对“无感填写”的期待推动系统向智能化、自动化演进  
  - [ ] 对应心理学原理: 控制感缺失引发的挫败感
